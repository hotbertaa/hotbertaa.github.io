Event Planner | Case Study

A social platform for organising events

Geronigo's event planner is a space for event organisers to plan and complete their event post enquiry / booking. Event participants can be invited and prompted for rsvp status. Individual guest payments are available, in addition to information on the event and a chat area.

The event planner is a valuable tool for managing booking payments & in retaining general event enquiries. It also helps extend Geronigo's reach beyond the event organiser; as one event could produce from 1 to 10's of new customers, extending the brands awareness and creating new revenue streams. How did I harness this?

Pros

Beyond the visual appeal of the new redesign, putting the focus on the guest list and its use in managing individual payments has given the event user a valuable tool and reason for using the event planner, aiding its ability to attract multiple users.

Cons

With more time I would like to add further confirmation pages on changes in state such as RSVP, payment confirmation. I'd also like to further the transitioning scope with principle.

Features from the development

Here are a few examples of key features implemented in the renewal prototype of Geronigo's event planner

Individual guest payments are invaluable to event organisers

With user feedback we learnt that most event organisers found the task of accumilating guest payments the most prohibitive reason for completing a booking and taking part in their event.

A heavily featured individual guest payment section was created and incorporated with the primary guest list. The payment section was a simple and efficient UI.

Post event feedback and photo's help Geronigo grow

User reviews are a useful source of SEO content and also improve customer confidence. Photo's are also very beneficial as it has often proven difficult to attain high quality venue imagery.

To encourage people to take the time to leave their feadback and photos, attention was paid to promote offers throughout the website and at key stages of the event timeline.

Notes from the beginning...

The concept was to build upon previous iterations of the event planner and create a contemporary mobile first UI.

Making it a valuable tool for the event organiser

The event planner needed more than event details and directions to encourage user engagement. How did I accomplish this?

  • check
    Giving the event organiser an individual guest payment feature.
  • check
    Promoting offers and discounts for their event.
  • check
    promoting the 'put it in your pocket' slogan by improving the mobile UI.

A modern mobile first user interface

The new prototype for the event planner needed to focus on the modern users need for a mobile first experience. How did I acheive this?

  • check
    Reassessed the hierachy of features & order of content.
  • check
    with the use of link transitions I aimed to create an innately intuitive UI.
  • check
    prompting for push notification approval, to allow direct messaging on a users phone.

Extend Geronigo's reach beyond the event organiser

One event could produce from 1 to 10's of new customers, extending the brands awareness and creating new revenue streams. How did I harness this?

  • check
    Offering individual online guest payments gave the event particpants a valuable reason to join the event planner.
  • check
